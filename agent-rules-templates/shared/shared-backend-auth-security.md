# 通用后端鉴权与安全规则

本文件描述后端认证、授权、租户隔离和审计底线。具体身份体系、权限模型、密钥来源和审计字段由项目级规则定义。

## 1. 认证

- 所有受保护接口必须经过统一认证入口，不得在业务代码中零散解析凭证。
- token、session、API key、签名和内部调用身份应明确来源、有效期、刷新、吊销和失败语义。
- 认证失败与权限不足必须区分，避免把身份缺失、身份过期和无权访问混为同一种错误。

## 2. 授权

- 权限判断应在服务端强制执行，前端展示控制不能替代后端鉴权。
- 权限规则应靠近业务资源和操作语义，避免只用路由名、菜单名或前端传参决定权限。
- 批量操作、导出、后台任务和异步回调也必须执行等价权限或来源校验。

## 3. 租户与数据隔离

- 多租户、组织、项目、门店、账号等隔离维度必须由服务端可信上下文决定。
- 查询、更新、删除、导出和统计必须带隔离条件，不能依赖客户端传入的租户标识。
- 跨租户管理能力必须有独立授权、审计和最小范围控制。

## 4. 敏感数据

- 密码、密钥、token、证件号、手机号、邮箱、地址、支付信息和内部配置默认视为敏感数据。
- 敏感数据不得进入 URL、普通日志、错误响应、指标标签、队列消息、缓存 key 和公开文档。
- 存储敏感数据时应明确加密、哈希、脱敏、访问控制、保留周期和清理策略。

## 5. 审计

- 登录、权限变更、资金、订单、审批、导出、删除、配置修改和越权失败应按项目规则记录审计。
- 审计日志应包含操作者、资源、动作、结果、时间、请求标识和必要业务上下文。
- 审计记录不得依赖客户端不可验证字段作为唯一事实来源。
