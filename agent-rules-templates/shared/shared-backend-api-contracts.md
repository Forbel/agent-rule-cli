# 通用后端 API 与契约规则

本文件描述后端接口、服务边界和错误响应底线。具体框架、路由、中间件、DTO、响应格式和版本策略由项目级规则定义。

## 1. 接口边界

- 外部入口应经过统一路由、中间件、参数校验、鉴权和错误处理链路。
- Controller / handler 负责协议适配，业务规则优先下沉到 service、domain 或 use case 层。
- 不得让数据库实体、ORM 对象或内部异常结构直接泄露为外部 API 契约。

## 2. 入参与响应

- 请求参数必须按来源区分 path、query、body、header、cookie 和上下文注入数据。
- 入参校验应覆盖类型、必填、长度、范围、格式、枚举、跨字段约束和权限相关约束。
- 响应字段、分页、排序、过滤、时间格式、金额精度和空值语义应遵守项目契约。
- 新增或变更外部 API 时，应说明兼容性、版本、灰度或调用方迁移策略。

## 3. 错误响应

- 错误响应应有稳定结构，至少区分参数错误、认证失败、权限不足、资源不存在、冲突、限流、业务失败和服务端异常。
- 对外错误信息不得暴露 SQL、堆栈、内部路径、密钥、权限细节或第三方原始错误。
- 内部日志可以记录脱敏诊断上下文，但对外响应必须保持可控且一致。

## 4. 幂等与并发

- 创建、支付、提交、审批、回调、导入和状态流转应按风险定义幂等键、唯一约束或去重策略。
- 并发写入必须明确依赖数据库约束、事务、乐观锁、悲观锁、分布式锁或业务状态机。
- 重试安全性必须由接口幂等性和副作用边界共同决定，不能只依赖客户端约定。

## 5. 调用下游

- 调用第三方或内部服务时，应定义超时、重试、熔断、降级、错误映射和链路追踪。
- 不得把下游临时异常直接升级为不可恢复业务状态。
- 高风险下游调用应保留请求标识、业务标识和脱敏错误上下文。
