# 通用后端可观测性规则

本文件描述后端日志、指标、链路追踪和诊断底线。具体日志库、trace header、指标平台和告警规则由项目级规则定义。

## 1. 日志

- 日志应记录请求标识、用户或系统身份、资源、动作、结果、耗时和必要业务上下文。
- 错误日志必须包含可定位原因的脱敏上下文，避免只记录“失败”或吞掉异常。
- 不得记录完整凭证、密钥、隐私、支付数据、大型响应体和不可公开的内部配置。

## 2. 指标

- 核心接口、任务、下游调用和数据库访问应关注吞吐、错误率、延迟、超时、重试和资源使用。
- 指标标签不得包含高基数字段、敏感数据或未脱敏业务明细。
- 性能优化应优先基于真实指标、慢查询、profile 或链路追踪证据。

## 3. 链路追踪

- 跨服务调用、异步任务和消息消费应尽量传递请求标识或 trace 上下文。
- 日志、指标、错误响应和审计记录中的请求标识应能互相关联。
- 调用下游时应保留服务名、接口、耗时、状态和脱敏错误分类。

## 4. 告警与诊断

- 登录、支付、订单、审批、权限、导入导出、消息积压和核心定时任务应定义异常诊断入口。
- 告警应区分用户影响、业务影响和系统影响，避免只有噪音没有行动路径。
- 交付时如无法完成验证或观测接入，应记录未覆盖范围和剩余风险。
