# 通用后端任务与消息规则

本文件描述队列、定时任务、异步事件和补偿流程底线。具体消息组件、任务框架、主题、重试次数和调度策略由项目级规则定义。

## 1. 任务边界

- 异步任务必须说明触发来源、输入、幂等键、执行方、超时、重试和最终状态。
- 定时任务应明确调度频率、时区、互斥策略、补跑策略和错过执行时的处理方式。
- 长耗时任务应避免阻塞请求线程，并提供可诊断的任务状态或执行记录。

## 2. 消息语义

- 消息 topic、event type、payload、版本、生产者、消费者和兼容性必须明确。
- 消费者不得假设消息只会送达一次；重复、乱序、延迟和过期消息都应有处理策略。
- 消息体不得携带不必要的敏感数据、大型 payload 或不可验证的权限上下文。

## 3. 重试与死信

- 重试必须区分临时错误、永久错误和业务拒绝，不能无限重试。
- 死信、失败表或补偿队列应保留足够脱敏上下文，支持人工诊断和安全重放。
- 重放任务或消息前必须确认幂等性、副作用边界和当前业务状态。

## 4. 一致性与补偿

- 本地事务与消息发送需要明确一致性方案，例如 outbox、事务消息、状态机或补偿任务。
- 外部系统调用失败时，应定义可恢复状态、补偿动作和人工介入条件。
- 不得把“任务已提交”误写成“业务已完成”。

## 5. 监控

- 核心任务和消费者应记录执行次数、成功率、失败率、耗时、积压、重试和死信指标。
- 高风险任务失败、积压异常和重复失败应触发告警或交付记录中的风险说明。
